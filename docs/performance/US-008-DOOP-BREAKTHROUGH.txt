NEXT STEPS (Phase 2D)
---------------------
K-FUSION infrastructure is correct and complete. To achieve the DOOP
breakthrough, the following additional optimizations are needed:

1. Workqueue parallel dispatch — activate multi-rule parallel evaluation
   so K-sequences run concurrently across worker threads. This is the
   primary missing piece; K-FUSION was designed to feed this path.

2. Demand-driven / semi-naive pruning — avoid re-deriving tuples already
   in the IDB; verify that delta tracking is tight for large recursive rules.

3. Relation-size-aware join ordering — for DOOP's call-graph construction
   rules, join order has order-of-magnitude impact; static or adaptive
   ordering could recover significant time.

4. Incremental index construction — avoid rebuilding sorted indexes on
   every iteration for stable EDB relations.
